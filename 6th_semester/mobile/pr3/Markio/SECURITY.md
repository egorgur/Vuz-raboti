# Средства обеспечения безопасности приложения Markio

Безопасность данных пользователей и стабильность работы приложения обеспечиваются следующими программными и архитектурными средствами:

## 1. Защита сетевого взаимодействия
*   **HTTPS/TLS:** Все запросы к API (погода, будущая синхронизация) выполняются исключительно через защищенный протокол HTTPS. Это предотвращает атаки типа "человек посередине" (MITM) и перехват данных.
*   **Network Security Configuration:** В приложении используется стандартная конфигурация безопасности сети Android, запрещающая передачу данных в открытом виде (cleartext traffic).

## 2. Защита кода и данных
*   **R8 / ProGuard:** В релизных сборках активирована обфускация (запутывание) кода. Это затрудняет декомпиляцию приложения и анализ его бизнес-логики злоумышленниками.
*   **Изоляция данных:** База данных Room хранится во внутреннем защищенном хранилище приложения, к которому другие приложения не имеют доступа (Sandbox).

## 3. Планируемые улучшения (Roadmap по безопасности)
*   **SQLCipher:** Внедрение шифрования базы данных на лету для защиты локальных заметок в случае получения root-прав на устройстве.
*   **EncryptedSharedPreferences:** Использование Jetpack Security для шифрования ключей авторизации и настроек пользователя.
*   **Биометрическая аутентификация:** Возможность входа в раздел "Заметки" по отпечатку пальца или Face ID.

## 4. Разрешения (Permissions)
Приложение придерживается принципа наименьших привилегий:
*   `INTERNET` — только для загрузки погоды.
*   `ACCESS_COARSE_LOCATION` — только для определения города пользователя (без доступа к точным координатам в фоновом режиме).
