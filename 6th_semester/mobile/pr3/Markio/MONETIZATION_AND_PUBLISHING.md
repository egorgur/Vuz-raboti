# План монетизации и публикации приложения Markio

## 1. План монетизации

Для приложения Markio выбрана гибридная модель монетизации:

### А. Модель Freemium (Основная)
*   **Бесплатные функции:**
    *   Создание и редактирование до 50 локальных заметок.
    *   Просмотр текущей погоды и карты в реальном времени.
*   **Платные функции (Premium подписка):**
    *   **Облачная синхронизация:** Доступ к заметкам с разных устройств.
    *   **Расширенная погода:** Прогноз на 7-14 дней и детализированные графики.
    *   **Кастомизация:** Дополнительные темы оформления и шрифты.
    *   **Отключение рекламы.**

### Б. Рекламная модель
*   Использование баннерной рекламы в нижней части экрана "Погода".
*   Межстраничная реклама (Interstitial) при частом переключении между экранами (только для бесплатных пользователей).

---

## 2. План публикации

### Этап 1: Подготовка
*   Создание политики конфиденциальности (Privacy Policy).
*   Подготовка графических материалов:
    *   Иконка (512x512, PNG).
    *   Скриншоты для телефонов и планшетов (минимум 4 для каждого типа).
    *   Баннер для страницы приложения (1024x500).

### Этап 2: Техническая подготовка
*   Обновление `versionCode` и `versionName` в `build.gradle.kts`.
*   Сборка подписанного **Android App Bundle (.aab)** вместо APK.

### Этап 3: Площадки
*   **Google Play Store:** Основная площадка для глобального рынка.
*   **RuStore:** Приоритетная площадка для российских пользователей.
*   **Huawei AppGallery:** Дополнительная площадка для устройств без Google-сервисов.
