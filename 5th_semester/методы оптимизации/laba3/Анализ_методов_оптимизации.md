## 🔬 Применённые методы оптимизации

### 1. Классический метод Ньютона (Newton's Method)

**Описание:** Метод Ньютона использует аналитические производные (градиент и матрицу Гессе) для нахождения минимума функции.

#### Математическая основа

Метод основан на разложении функции в ряд Тейлора до второго порядка:

```
f(x + p) ≈ f(x) + ∇f(x)ᵀp + ½pᵀH(x)p
```

где:
- `∇f(x)` — градиент функции
- `H(x)` — матрица Гессе (гессиан)
- `p` — направление поиска

#### Алгоритм работы (пошагово)

**Шаг 1: Инициализация**
- Задаётся начальная точка `x₀`
- Устанавливается точность `ε` (по умолчанию 1e-6)
- Устанавливается максимальное число итераций `max_iter` (по умолчанию 1000)

**Шаг 2: Вычисление градиента и гессиана**
- Вычисляется градиент: `∇f(xₖ)`
- Вычисляется матрица Гессе: `H(xₖ)`

Для функции **f1**:
- Градиент: `∇f1 = [2(x₁ - 4), 2(x₂ - 1)]`
- Гессиан: `H₁ = [[2, 0], [0, 2]]` (постоянная матрица)

Для функции **f2**:
- Градиент: `∇f2 = [8x₁ + x₂, x₁ + 4x₂ - 2]`
- Гессиан: `H₂ = [[8, 1], [1, 4]]` (постоянная матрица)

**Шаг 3: Проверка условия остановки**
- Вычисляется норма градиента: `||∇f(xₖ)||`
- Если `||∇f(xₖ)|| < ε`, алгоритм завершается (достигнута стационарная точка)

**Шаг 4: Решение системы линейных уравнений**
- Решается система: `H(xₖ) · p = -∇f(xₖ)`
- Находится направление поиска `p`
- Если матрица Гессе вырождена, используется псевдообратная матрица

**Шаг 5: Обновление точки**
- Новая точка: `xₖ₊₁ = xₖ + p`
- Вычисляется значение функции в новой точке: `f(xₖ₊₁)`

**Шаг 6: Дополнительная проверка сходимости**
- Если `||xₖ₊₁ - xₖ|| < ε`, алгоритм завершается

**Шаг 7: Итерация**
- Переход к следующей итерации: `k = k + 1`
- Повторение шагов 2-6 до достижения условия остановки

#### Особенности реализации

```python
def newton_method(f, grad_f, hessian_f, x0, eps=1e-6, max_iter=1000):
    x_current = x0.copy()
    
    for k in range(max_iter):
        grad = grad_f(x_current)      # Аналитический градиент
        hessian = hessian_f(x_current) # Аналитический гессиан
        
        if np.linalg.norm(grad) < eps:
            break
        
        p = np.linalg.solve(hessian, -grad)  # Решение системы
        x_new = x_current + p
        
        if np.linalg.norm(x_new - x_current) < eps:
            break
            
        x_current = x_new
```

#### Преимущества
- ✅ **Квадратичная сходимость** для хорошо обусловленных функций
- ✅ **Минимальное количество вычислений** функции (только в новой точке)
- ✅ **Высокая точность** при наличии аналитических производных

#### Недостатки
- ❌ Требует аналитических производных
- ❌ Может быть неустойчив при плохо обусловленной матрице Гессе
- ❌ Не гарантирует сходимость к глобальному минимуму

### 2. Метод Ньютона-Рафсона (Newton-Raphson Method)

**Описание:** Метод Ньютона-Рафсона использует численное дифференцирование для вычисления производных, что делает его более универсальным.

#### Математическая основа

Метод использует те же принципы, что и классический метод Ньютона, но производные вычисляются численно через конечные разности.

#### Алгоритм работы (пошагово)

**Шаг 1: Инициализация**
- Задаётся начальная точка `x₀`
- Устанавливается точность `ε` (по умолчанию 1e-6)
- Устанавливается шаг численного дифференцирования `h` (по умолчанию 1e-6)
- Устанавливается максимальное число итераций `max_iter`

**Шаг 2: Численное вычисление градиента**

Для каждой компоненты `i` градиента используется центральная разность:

```
∂f/∂xᵢ ≈ [f(x + h·eᵢ) - f(x - h·eᵢ)] / (2h)
```

где `eᵢ` — единичный вектор в направлении `i`-й координаты.

**Алгоритм:**
```python
def numerical_gradient(x):
    grad = np.zeros_like(x)
    for i in range(len(x)):
        x_plus = x.copy()
        x_minus = x.copy()
        x_plus[i] += h
        x_minus[i] -= h
        grad[i] = (f(x_plus) - f(x_minus)) / (2 * h)
    return grad
```

**Количество вычислений функции:** `2n` (где `n` — размерность)

**Шаг 3: Численное вычисление матрицы Гессе**

**Диагональные элементы** (вторые производные):
```
∂²f/∂xᵢ² ≈ [f(x + h·eᵢ) - 2f(x) + f(x - h·eᵢ)] / h²
```

**Внедиагональные элементы** (смешанные производные):
```
∂²f/∂xᵢ∂xⱼ ≈ [f(x + h·eᵢ + h·eⱼ) - f(x + h·eᵢ - h·eⱼ) 
              - f(x - h·eᵢ + h·eⱼ) + f(x - h·eᵢ - h·eⱼ)] / (4h²)
```

**Алгоритм:**
```python
def numerical_hessian(x):
    n = len(x)
    hessian = np.zeros((n, n))
    f_base = f(x)  # Базовое значение
    
    # Диагональные элементы
    for i in range(n):
        x_plus = x.copy()
        x_minus = x.copy()
        x_plus[i] += h
        x_minus[i] -= h
        hessian[i, i] = (f(x_plus) - 2*f_base + f(x_minus)) / (h**2)
    
    # Внедиагональные элементы
    for i in range(n):
        for j in range(i + 1, n):
            # 4 точки для смешанной производной
            x_pp = x.copy(); x_pp[i] += h; x_pp[j] += h
            x_pm = x.copy(); x_pm[i] += h; x_pm[j] -= h
            x_mp = x.copy(); x_mp[i] -= h; x_mp[j] += h
            x_mm = x.copy(); x_mm[i] -= h; x_mm[j] -= h
            
            hessian[i, j] = (f(x_pp) - f(x_pm) - f(x_mp) + f(x_mm)) / (4*h**2)
            hessian[j, i] = hessian[i, j]  # Симметрия
    
    return hessian
```

**Количество вычислений функции:** 
- Базовое значение: `1`
- Диагональные: `2n`
- Внедиагональные: `4·n(n-1)/2 = 2n(n-1)`
- **Итого на итерацию:** `1 + 2n + 2n(n-1) = 1 + 2n²`

Для `n=2`: `1 + 2·2² = 9` вычислений на итерацию (плюс вычисление в новой точке = 10)

**Шаг 4: Проверка условия остановки**
- Вычисляется норма численного градиента: `||∇f(xₖ)||`
- Если `||∇f(xₖ)|| < ε`, алгоритм завершается

**Шаг 5: Решение системы линейных уравнений**
- Решается система: `H(xₖ) · p = -∇f(xₖ)`
- Используется `np.linalg.solve()` или псевдообратная матрица при вырожденности

**Шаг 6: Обновление точки**
- Новая точка: `xₖ₊₁ = xₖ + p`
- Вычисляется значение функции: `f(xₖ₊₁)`

**Шаг 7: Дополнительная проверка сходимости**
- Если `||xₖ₊₁ - xₖ|| < ε`, алгоритм завершается

**Шаг 8: Итерация**
- Переход к следующей итерации
- Повторение шагов 2-7

#### Особенности реализации

```python
def newton_raphson_method(f, x0, eps=1e-6, max_iter=1000, h=1e-6):
    x_current = x0.copy()
    
    for k in range(max_iter):
        grad = numerical_gradient(x_current)    # Численный градиент
        hessian = numerical_hessian(x_current)  # Численный гессиан
        
        if np.linalg.norm(grad) < eps:
            break
        
        p = np.linalg.solve(hessian, -grad)
        x_new = x_current + p
        
        if np.linalg.norm(x_new - x_current) < eps:
            break
            
        x_current = x_new
```

#### Преимущества
- ✅ **Не требует аналитических производных** — универсален
- ✅ **Применим к любым функциям** (даже если производные неизвестны)
- ✅ **Квадратичная сходимость** при правильном выборе шага `h`

#### Недостатки
- ❌ **Большое количество вычислений функции** (O(n²) на итерацию)
- ❌ **Чувствительность к выбору шага `h`**:
  - Слишком большой `h` → большая ошибка численного дифференцирования
  - Слишком малый `h` → ошибки округления, нестабильность
- ❌ **Медленнее метода Ньютона** из-за численного дифференцирования

## 📈 Влияние параметров

### Влияние точности ε

**Метод Ньютона:**
- Уменьшение `ε` не влияет на количество итераций (для квадратичных функций всегда 1)
- Влияет только на проверку условия остановки

**Метод Ньютона-Рафсона:**
- Уменьшение `ε` может увеличить количество итераций
- Влияет на точность результата

### Влияние шага h (только для Ньютона-Рафсона)

**Оптимальный шаг:** `h ≈ 1e-4`

**Проблемы:**
- **Слишком большой `h` (≥ 1e-2):** Большая ошибка численного дифференцирования
- **Слишком малый `h` (≤ 1e-6):** Ошибки округления, нестабильность

**Рекомендация:** `h = ε^(1/3)` или `h = sqrt(ε)` для баланса между точностью и стабильностью

## 🔧 Технические детали реализации

### Класс CountedFunc

Используется для подсчёта количества вычислений целевой функции:

```python
class CountedFunc:
    def __init__(self, func):
        self.func = func
        self.evals = 0
    
    def __call__(self, x):
        self.evals += 1
        return self.func(x)
```

### Обработка вырожденных матриц

При вырожденной матрице Гессе используется псевдообратная матрица:

```python
try:
    p = np.linalg.solve(hessian, -grad)
except np.linalg.LinAlgError:
    p = -np.linalg.pinv(hessian) @ grad
```

### Условия остановки

1. **Норма градиента:** `||∇f(x)|| < ε`
2. **Изменение точки:** `||xₖ₊₁ - xₖ|| < ε`
3. **Максимальное число итераций:** `k ≥ max_iter`

## 📚 Математические основы

### Квадратичная сходимость

Метод Ньютона имеет квадратичную сходимость, если:
- Функция дважды дифференцируема
- Матрица Гессе положительно определена в окрестности минимума
- Начальная точка достаточно близка к минимуму

**Скорость сходимости:**
```
||xₖ₊₁ - x*|| ≤ C ||xₖ - x*||²
```

где `C` — константа, зависящая от третьих производных.

### Условия оптимальности

**Необходимое условие первого порядка:**
```
∇f(x*) = 0
```

**Достаточное условие второго порядка:**
```
∇f(x*) = 0  и  H(x*) положительно определена
```
