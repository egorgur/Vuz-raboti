## Применённые методы оптимизации

### Генетический алгоритм (Genetic Algorithm)

Генетический алгоритм — это эволюционный метод оптимизации, основанный на принципах естественного отбора и генетики. Алгоритм работает с популяцией решений (особей), которые эволюционируют через поколения.

---

## Детальное описание работы алгоритма по шагам

### Шаг 1: Кодирование решения (Representation)

**Бинарное кодирование переменных**

Каждая переменная кодируется в виде бинарной строки фиксированной длины.

**Процесс кодирования:**
1. Для каждой переменной `x_i` с границами `[x_min, x_max]` выделяется `bits_per_var` бит
2. Бинарная строка интерпретируется как целое число от 0 до `2^bits_per_var - 1`
3. Это число масштабируется в диапазон `[x_min, x_max]` по формуле:
   ```
   x_real = x_min + (x_max - x_min) * int_value / (2^bits_per_var - 1)
   ```

**Пример:**
- Переменная: x ∈ [-1, 1]
- Бит на переменную: 8
- Бинарная строка: `[1,0,1,1,0,0,1,0]` → целое: 178
- Реальное значение: -1 + (1 - (-1)) * 178 / 255 ≈ 0.396

**Хромосома (генотип):**
- Для 2 переменных с 8 битами каждая: хромосома = 16 бит
- Первые 8 бит → первая переменная
- Следующие 8 бит → вторая переменная

---

### Шаг 2: Инициализация популяции

**Процесс:**
1. Создаётся начальная популяция размером `pop_size`
2. Каждая особь (хромосома) генерируется случайно:
   - Для каждого бита случайно выбирается 0 или 1
   - Всего бит: `num_vars * bits_per_var`

**Пример:**
- Размер популяции: 50
- Количество переменных: 2
- Бит на переменную: 8
- Каждая хромосома: 16 бит
- Популяция: список из 50 хромосом по 16 бит

---

### Шаг 3: Оценка приспособленности (Fitness Evaluation)

**Процесс:**
1. Для каждой хромосомы в популяции:
   - **Декодирование**: бинарная строка → вещественные значения переменных
   - **Вычисление функции**: подстановка значений в целевую функцию
   - **Fitness**: значение функции (для минимизации: меньше = лучше)

**Формула декодирования:**
```python
def decode(chromosome):
    decoded = []
    for i in range(num_vars):
        start = i * bits_per_var
        end = start + bits_per_var
        bits = chromosome[start:end]
        int_value = int("".join(map(str, bits)), 2)
        x_min, x_max = bounds[i]
        real_value = x_min + (x_max - x_min) * int_value / (2^bits_per_var - 1)
        decoded.append(real_value)
    return decoded
```

**Результат:**
- Массив fitness-значений для всей популяции
- Лучшая особь: минимальное fitness-значение

### Шаг 4: Селекция (Selection) — отбор родителей

Алгоритм реализует три метода селекции:

#### 4.1. Ранговая селекция (Rank Selection)

**Принцип:**
- Особи сортируются по приспособленности (лучшие → худшие)
- Вероятность выбора зависит от ранга, а не от абсолютного значения fitness

**Алгоритм:**
1. Сортировка популяции по fitness (от лучшего к худшему)
2. Присвоение рангов: лучшая особь получает ранг N, худшая — ранг 1
3. Вычисление вероятностей:
   ```
   rank_i = N - i + 1  (для i-й особи после сортировки)
   prob_i = 2 * rank_i / (N * (N + 1))
   ```
4. Выбор родителей: случайный выбор с учётом вероятностей

**Преимущества:**
- Предотвращает доминирование супер-особей на ранних этапах
- Поддерживает разнообразие популяции

**Пример:**
- Популяция из 5 особей
- Ранги: [5, 4, 3, 2, 1]
- Вероятности: [0.333, 0.267, 0.200, 0.133, 0.067]

#### 4.2. Турнирная селекция (Tournament Selection)

**Принцип:**
- Случайно выбирается группа особей (турнир)
- Победитель турнира (лучшая особь) становится родителем

**Алгоритм:**
1. Для каждого родителя:
   - Случайный выбор `tournament_size` особей из популяции
   - Сравнение их fitness-значений
   - Выбор особи с минимальным fitness (для минимизации)
2. Повторение для всех необходимых родителей

**Параметры:**
- `tournament_size`: размер турнира (обычно 2-7)
- Чем больше размер, тем выше давление отбора

**Преимущества:**
- Простота реализации
- Эффективность вычислений
- Контролируемое давление отбора

**Пример:**
- Размер турнира: 3
- Выбраны особи: [A, B, C] с fitness [0.5, 0.2, 0.8]
- Победитель: B (минимальный fitness = 0.2)

#### 4.3. Пропорциональная селекция (Proportional Selection)

**Принцип:**
- Вероятность выбора пропорциональна приспособленности особи

**Алгоритм:**
1. Для задачи минимизации: инверсия fitness
   ```
   C = max(fitness) + ε  (константа для нормализации)
   prob_i = (-fitness_i + C) / (N*C - sum(fitness))
   ```
2. Нормализация вероятностей (сумма = 1)
3. Выбор родителей: случайный выбор с учётом вероятностей

**Преимущества:**
- Прямая связь между fitness и вероятностью выбора
- Хорошо работает для задач максимизации

**Недостатки:**
- При малых различиях в fitness вероятности почти равны
- При больших различиях доминируют лучшие особи

---

### Шаг 5: Кроссовер (Crossover) — скрещивание

Объединение генетического материала двух родителей для создания потомков.

**Процесс:**
1. Родители разбиваются на пары
2. С вероятностью `crossover_rate` применяется кроссовер
3. Если кроссовер не применяется, потомки = копии родителей

#### 5.1. Одноточечный кроссовер (One-Point Crossover)

**Алгоритм:**
1. Выбор случайной точки разрыва `k` (1 ≤ k < длина хромосомы)
2. Создание потомков:
   ```
   child1 = parent1[0:k] + parent2[k:end]
   child2 = parent2[0:k] + parent1[k:end]
   ```

**Пример:**
- Parent1: `[1,0,1,1,0,0,1,0]`
- Parent2: `[0,1,1,0,1,1,0,1]`
- Точка разрыва: k = 3
- Child1: `[1,0,1,0,1,1,0,1]`
- Child2: `[0,1,1,1,0,0,1,0]`

#### 5.2. Двухточечный кроссовер (Two-Point Crossover)

**Алгоритм:**
1. Выбор двух точек разрыва: `k1 < k2`
2. Создание потомков:
   ```
   child1 = parent1[0:k1] + parent2[k1:k2] + parent1[k2:end]
   child2 = parent2[0:k1] + parent1[k1:k2] + parent2[k2:end]
   ```

**Пример:**
- Parent1: `[1,0,1,1,0,0,1,0]`
- Parent2: `[0,1,1,0,1,1,0,1]`
- Точки: k1 = 2, k2 = 5
- Child1: `[1,0,1,0,1,1,1,0]`
- Child2: `[0,1,1,1,0,0,0,1]`

#### 5.3. Равномерный кроссовер (Uniform Crossover)

**Алгоритм:**
1. Для каждого бита:
   - С вероятностью 0.5: child1 берёт бит от parent1, child2 — от parent2
   - Иначе: child1 берёт бит от parent2, child2 — от parent1

**Пример:**
- Parent1: `[1,0,1,1]`
- Parent2: `[0,1,1,0]`
- Маска: `[0,1,0,1]` (0 = от parent1, 1 = от parent2)
- Child1: `[1,1,1,0]`
- Child2: `[0,0,1,1]`

**Параметры:**
- `crossover_rate`: вероятность применения кроссовера (обычно 0.6-0.9)

---

### Шаг 6: Мутация (Mutation)

Внесение случайных изменений в хромосомы для поддержания разнообразия.

**Алгоритм:**
1. Для каждой хромосомы в популяции:
2. Для каждого бита:
   - С вероятностью `mutation_rate`: инверсия бита (0 → 1, 1 → 0)
   - Иначе: бит остаётся без изменений

**Сила мутации:**
- **Слабая (weak)**: `mutation_rate` (обычно 1/64 ≈ 0.0156)
- **Сильная (strong)**: `mutation_rate * 5` (обычно 5/64 ≈ 0.078)

**Пример:**
- Хромосома: `[1,0,1,1,0,0,1,0]`
- Мутация бита 3: `[1,0,1,0,0,0,1,0]`

**Роль мутации:**
- Восстановление потерянного генетического материала
- Исследование новых областей пространства поиска
- Предотвращение преждевременной сходимости

---

### Шаг 7: Элитизм (Elitism)

**Принцип:**
- Лучшие особи текущего поколения автоматически переходят в следующее

**Алгоритм:**
1. Сортировка текущей популяции по fitness
2. Выбор `elite_size` лучших особей
3. Эти особи добавляются в новое поколение без изменений
4. Остальные места заполняются потомками

**Преимущества:**
- Гарантирует, что лучшее решение не потеряется
- Ускоряет сходимость алгоритма

**Недостатки:**
- Может привести к преждевременной сходимости при большом `elite_size`
- Снижает разнообразие популяции

**Параметры:**
- `elite_size`: количество элитных особей (обычно 1-5% от размера популяции)

---

### Шаг 8: Формирование нового поколения

**Процесс:**
1. Элитные особи: `elite_size` лучших из текущей популяции
2. Потомки: случайная выборка из созданных потомков
3. Новое поколение = элитные + потомки (до размера `pop_size`)

**Важно:**
- Размер популяции остаётся постоянным
- Если потомков больше, чем нужно, выбираются случайные

---

### Шаг 9: Основной эволюционный цикл

**Алгоритм работы ГА:**

```
1. Инициализация популяции
2. Для каждого поколения (generations):
   a. Оценка приспособленности всех особей
   b. Сохранение статистики (лучшее, среднее, худшее)
   c. Селекция родителей
   d. Кроссовер → создание потомков
   e. Мутация потомков
   f. Формирование нового поколения (элитизм + потомки)
3. Возврат лучшего решения
```

**Критерии остановки:**
- Достижение максимального числа поколений
- Достижение целевого значения функции
- Отсутствие улучшений в течение N поколений

---

## Метрики оценки алгоритма

### Надёжность (Reliability)
```
Надёжность = (Количество успешных запусков / Общее количество запусков) * 100%
```
- **Успешный запуск**: найденное решение находится в пределах `[target - limit, target + limit]`
- **Limit** зависит от функции:
  - Ф1, Ф3, Ф12: 0.003
  - Ф8 (Sombrero): 0.00000003
  - Ф5 (Rosenbrock): 0.04

### Среднее число итераций (Average Iterations)
```
Среднее число итераций = Среднее значение номера поколения, 
                          когда найдено решение (по успешным запускам)
```

**Критерии сравнения:**
1. Алгоритм с наибольшей надёжностью считается лучшим
2. При одинаковой надёжности лучший — с наименьшим средним числом итераций

## Параметры алгоритма

### Основные параметры:

1. **pop_size** (размер популяции): 30-50
   - Больше → лучшее исследование, но медленнее
   - Меньше → быстрее, но риск преждевременной сходимости

2. **generations** (количество поколений): 50-200
   - Больше → больше шансов найти решение
   - Меньше → быстрее выполнение

3. **bits_per_var** (бит на переменную): 8-64
   - Больше → выше точность, но больше размер хромосомы
   - Меньше → быстрее, но ниже точность

4. **crossover_rate** (вероятность кроссовера): 0.5-0.9
   - Высокая → больше обмена генетическим материалом
   - Низкая → больше копий родителей

5. **mutation_rate** (вероятность мутации): 1/64 - 1/32
   - Высокая → больше разнообразия, но может разрушить хорошие решения
   - Низкая → стабильность, но риск застревания в локальных минимумах

6. **elite_size** (количество элитных особей): 1-5% от pop_size
   - Больше → быстрее сходимость, но меньше разнообразие
   - Меньше → больше разнообразия, но медленнее сходимость

7. **tournament_size** (размер турнира): 2-7
   - Больше → сильнее давление отбора
   - Меньше → слабее давление отбора

## Структура реализации

### Класс `GeneticAlgorithm` (gen.py)

**Основные методы:**
- `initialize_population()`: создание начальной популяции
- `decode()`: преобразование бинарной строки в вещественные значения
- `evaluate()`: вычисление fitness для всех особей
- `selection()`: отбор родителей (rank/tournament/proportional)
- `crossover()`: скрещивание (one_point/two_point/uniform)
- `mutate()`: мутация (weak/strong)
- `form_new_population()`: формирование нового поколения с элитизмом
- `run()`: основной цикл алгоритма

### Класс `GeneticAlgorithmExperiments` (experiment.py)

**Основные методы:**
- `run_experiments()`: запуск серии экспериментов с разными конфигурациями
- `compare_results()`: сравнение результатов и построение графиков
- Вычисление метрик: надёжность, среднее число итераций

### GUI интерфейс (main.py)

**Функциональность:**
- Выбор тестовой функции
- Настройка параметров алгоритма
- Выбор методов селекции, кроссовера, мутации
- Запуск экспериментов
- Отображение результатов в таблице
- Построение графиков сходимости
