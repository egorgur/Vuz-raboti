## 1. Метод золотого сечения (Golden Section Search)

### Описание
Вспомогательный метод для одномерной оптимизации, используемый в качестве подзадачи для поиска оптимального шага в многомерных методах.

### Принцип работы

**Шаг 1: Инициализация**
- Задаются границы интервала `[a, b]`, в котором находится минимум
- Вычисляется коэффициент золотого сечения: `φ = (√5 - 1)/2 ≈ 0.618`
- Вычисляются две внутренние точки:
  - `c = b - φ(b - a)`
  - `d = a + φ(b - a)`
- Вычисляются значения функции: `f(c)` и `f(d)`

**Шаг 2: Итерационный процесс**
На каждой итерации:
1. **Сравнение значений**: Если `f(c) < f(d)`:
   - Новый интервал: `[a, d]` (правая граница сдвигается влево)
   - `b = d`, `d = c`, `f(d) = f(c)`
   - Вычисляется новая точка: `c = b - φ(b - a)`, `f(c) = f(c)`
   
   Иначе:
   - Новый интервал: `[c, b]` (левая граница сдвигается вправо)
   - `a = c`, `c = d`, `f(c) = f(d)`
   - Вычисляется новая точка: `d = a + φ(b - a)`, `f(d) = f(d)`

2. **Проверка сходимости**: Если `|b - a| < ε`, процесс останавливается

**Шаг 3: Результат**
- Возвращается середина финального интервала: `x_min = (a + b)/2`
- Значение функции в этой точке: `f_min = f(x_min)`

### Преимущества
- Не требует вычисления производных
- Гарантированная сходимость для унимодальных функций
- Стабильная работа

### Недостатки
- Работает только для одномерных функций
- Медленнее методов, использующих производные

## 2. Метод наискорейшего градиентного спуска (Gradient Descent with Optimal Step)

### Описание
Классический метод первого порядка, который на каждой итерации движется в направлении антиградиента с оптимальным шагом.

### Принцип работы по шагам

**Шаг 1: Инициализация**
- Задается начальная точка `x₀`
- Устанавливается точность `ε` и максимальное число итераций

**Шаг 2: Итерационный процесс**

Для каждой итерации `k = 0, 1, 2, ...`:

1. **Вычисление градиента**:
   - Используется численное дифференцирование:
     ```
     ∇f(x)[i] ≈ (f(x + h·eᵢ) - f(x - h·eᵢ)) / (2h)
     ```
     где `eᵢ` - единичный вектор по i-й координате, `h = 10⁻⁶`

2. **Проверка условия остановки**:
   - Если `||∇f(xₖ)|| < ε`, алгоритм останавливается
   - Это означает, что достигнута стационарная точка

3. **Определение направления спуска**:
   - Направление: `dₖ = -∇f(xₖ)` (антиградиент)

4. **Поиск оптимального шага** (одномерная оптимизация):
   - Решается задача: `minimize φ(α) = f(xₖ + α·dₖ)` по `α ≥ 0`
   - Используется метод золотого сечения на интервале `[0, 1]`
   - Находится оптимальный шаг `αₖ`

5. **Обновление точки**:
   - `xₖ₊₁ = xₖ + αₖ·dₖ`

6. **Дополнительная проверка сходимости**:
   - Если `||xₖ₊₁ - xₖ|| < ε`, алгоритм останавливается

**Шаг 3: Результат**
- Возвращается `x_min = xₖ` (последняя точка)
- Значение функции `f_min = f(x_min)`
- Количество итераций и вычислений функции

### Математическое обоснование

Градиент указывает направление наискорейшего роста функции. Антиградиент `-∇f(x)` указывает направление наискорейшего убывания. Оптимальный шаг `αₖ` находится из условия:
```
αₖ = argmin_{α≥0} f(xₖ + α·dₖ)
```

### Преимущества
- Простота реализации
- Гарантированная сходимость для выпуклых функций
- Не требует вычисления матрицы Гессе

### Недостатки
- Медленная сходимость для плохо обусловленных задач (эллипсоиды с большим числом обусловленности)
- "Зигзагообразное" движение в узких долинах
- Для квадратичных функций сходимость может быть медленной

### Особенности реализации
- Используется численное дифференцирование для вычисления градиента
- Оптимальный шаг находится методом золотого сечения
- Подсчитывается количество вычислений функции

## 3. Метод Флетчера-Ривса (Fletcher-Reeves Method)

### Описание
Метод сопряженных градиентов, который использует информацию о предыдущих направлениях для построения сопряженных направлений. Для квадратичных функций теоретически сходится за `n` итераций (где `n` - размерность задачи).

### Принцип работы по шагам

**Шаг 1: Инициализация**
- Задается начальная точка `x₀`
- Вычисляется начальный градиент: `g₀ = ∇f(x₀)`
- Начальное направление: `d₀ = -g₀` (антиградиент)

**Шаг 2: Итерационный процесс**

Для каждой итерации `k = 0, 1, 2, ...`:

1. **Поиск оптимального шага**:
   - Решается одномерная задача: `minimize φ(α) = f(xₖ + α·dₖ)`
   - Используется метод золотого сечения
   - Находится оптимальный шаг `αₖ`

2. **Обновление точки**:
   - `xₖ₊₁ = xₖ + αₖ·dₖ`

3. **Вычисление нового градиента**:
   - `gₖ₊₁ = ∇f(xₖ₊₁)`

4. **Вычисление коэффициента Флетчера-Ривса**:
   ```
   βₖ = (gₖ₊₁ᵀ · gₖ₊₁) / (gₖᵀ · gₖ)
   ```
   Это отношение квадратов норм градиентов

5. **Построение нового сопряженного направления**:
   ```
   dₖ₊₁ = -gₖ₊₁ + βₖ·dₖ
   ```
   Новое направление является линейной комбинацией антиградиента и предыдущего направления

6. **Проверка условия остановки**:
   - Если `||gₖ₊₁|| < ε`, алгоритм останавливается

7. **Обновление переменных**:
   - `xₖ = xₖ₊₁`
   - `gₖ = gₖ₊₁`
   - `dₖ = dₖ₊₁`

**Шаг 3: Результат**
- Возвращается найденный минимум и статистика работы алгоритма

### Математическое обоснование

**Сопряженность направлений**:
Два направления `dᵢ` и `dⱼ` называются сопряженными относительно матрицы `A`, если:
```
dᵢᵀ · A · dⱼ = 0,  i ≠ j
```

Для квадратичной функции `f(x) = ½xᵀAx + bᵀx + c`, где `A` - матрица Гессе, метод строит последовательность сопряженных направлений. Это позволяет за `n` итераций достичь минимума (теоретически).

**Коэффициент Флетчера-Ривса** обеспечивает сопряженность направлений для квадратичных функций.

### Преимущества
- **Быстрая сходимость**: Для квадратичных функций сходится за `n` итераций (теоретически)
- **Эффективность**: Требует меньше итераций, чем градиентный спуск
- **Не требует матрицы Гессе**: Использует только градиенты
- **Конечная сходимость**: Для квадратичных функций гарантированно находит минимум за конечное число шагов

### Недостатки
- Эффективность снижается для нелинейных функций
- Требует перезапуска (reset) для нелинейных задач
- Чувствителен к ошибкам округления

### Особенности реализации
- Используется формула Флетчера-Ривса для вычисления `β`
- Оптимальный шаг находится методом золотого сечения
- Для квадратичных функций работает особенно эффективно

## 4. Метод Гаусса-Зейделя (Gauss-Seidel Method, Coordinate Descent)

### Описание
Метод покоординатного спуска, который минимизирует функцию по одной координате за раз, фиксируя остальные координаты.

### Принцип работы по шагам

**Шаг 1: Инициализация**
- Задается начальная точка `x₀`
- Задаются границы для каждой координаты: `bounds[i] = [aᵢ, bᵢ]`
- Устанавливается точность `ε`

**Шаг 2: Итерационный процесс**

Для каждой итерации `k = 0, 1, 2, ...`:

1. **Сохранение текущей точки**: `x_old = xₖ` (копия)

2. **Цикл по координатам** (для каждой координаты `i = 0, 1, ..., n-1`):

   a. **Создание одномерной функции**:
      ```
      φᵢ(xᵢ) = f(x₀, x₁, ..., xᵢ, ..., xₙ₋₁)
      ```
      Все координаты, кроме `i`-й, фиксированы на текущих значениях
   
   b. **Минимизация по i-й координате**:
      - Используется метод золотого сечения на интервале `[bounds[i][0], bounds[i][1]]`
      - Находится оптимальное значение: `xᵢ* = argmin φᵢ(xᵢ)`
   
   c. **Обновление координаты**:
      - `xₖ[i] = xᵢ*`

3. **Проверка условия остановки**:
   - Если `||xₖ - x_old|| < ε`, алгоритм останавливается
   - Это означает, что все координаты стабилизировались

**Шаг 3: Результат**
- Возвращается найденный минимум и статистика

### Математическое обоснование

На каждой итерации решается последовательность одномерных задач оптимизации:
```
xₖ⁽⁰⁾ = xₖ
xₖ⁽¹⁾ = argmin_{x₀} f(x₀, xₖ,₁, ..., xₖ,ₙ₋₁)
xₖ⁽²⁾ = argmin_{x₁} f(xₖ⁽¹⁾₀, x₁, xₖ,₂, ..., xₖ,ₙ₋₁)
...
xₖ₊₁ = argmin_{xₙ₋₁} f(xₖ⁽ⁿ⁻¹⁾₀, ..., xₙ₋₁)
```

### Преимущества
- **Простота**: Каждая подзадача - одномерная оптимизация
- **Надежность**: Работает даже когда градиент вычислить сложно
- **Параллелизация**: Можно минимизировать по разным координатам параллельно (с модификациями)
- **Устойчивость**: Менее чувствителен к плохой обусловленности

### Недостатки
- **Медленная сходимость**: Может быть очень медленным для плохо обусловленных задач
- **Зависимость от системы координат**: Результаты зависят от выбора базиса
- **Много вычислений**: Требует много вычислений функции (на каждую координату на каждой итерации)

### Особенности реализации
- Используется метод золотого сечения для каждой координаты
- Требует задания границ для каждой координаты
- Подсчитывается общее количество вычислений функции

## Сравнение методов

### Теоретические характеристики

| Метод | Порядок сходимости | Требования | Сложность итерации |
|-------|-------------------|------------|-------------------|
| Градиентный спуск | Линейная | Градиент | O(n) |
| Флетчера-Ривса | Конечная (для квадратичных) | Градиент | O(n) |
| Гаусса-Зейделя | Линейная | Только функция | O(n) |

### Практические результаты (из кода)

**Для функции Гаусса:**
- **Флетчера-Ривса**: 0-3 итерации, 2-5 вычислений функции
- **Градиентный спуск**: 0-5 итераций, 1-5 вычислений функции
- **Гаусса-Зейделя**: 1-6 итераций, 34-432 вычислений функции

**Для квадратичной функции:**
- **Флетчера-Ривса**: Наиболее эффективен
- **Градиентный спуск**: Хорошая сходимость
- **Гаусса-Зейделя**: Больше вычислений функции

### Рекомендации по выбору метода

1. **Для квадратичных функций**: Метод Флетчера-Ривса - оптимальный выбор
2. **Для общих выпуклых функций**: Градиентный спуск - хороший баланс простоты и эффективности
3. **Когда градиент недоступен или сложен**: Метод Гаусса-Зейделя - надежный вариант
4. **Для больших размерностей**: Флетчера-Ривса предпочтительнее из-за конечной сходимости

## Целевые функции

### 1. Функция Гаусса: `f(x) = x₁² + (4x₁ + x₂)² + 18x₂²`

**Свойства:**
- Минимум: `x* = (0, 0)`, `f(x*) = 0`
- Матрица Гессе: `H = [[34, 8], [8, 38]]`
- Собственные значения: `λ₁ ≈ 39.66`, `λ₂ ≈ 32.34`
- Число обусловленности: `κ ≈ 1.23` (хорошо обусловлена)
- Сильно выпуклая квадратичная функция

**Раскрытие скобок:**
```
f(x) = x₁² + (4x₁ + x₂)² + 18x₂²
     = x₁² + 16x₁² + 8x₁x₂ + x₂² + 18x₂²
     = 17x₁² + 8x₁x₂ + 19x₂²
```

### 2. Квадратичная функция: `f(x) = 2x₁² - 2x₁ + x₁x₂ - x₂ + x₂²`

**Свойства:**
- Минимум: `x* = (3/7, 2/7) ≈ (0.4286, 0.2857)`, `f(x*) = -5/7 ≈ -0.7143`
- Матрица Гессе: `H = [[4, 1], [1, 2]]`
- Собственные значения: `λ₁ ≈ 4.41`, `λ₂ ≈ 1.59`
- Число обусловленности: `κ ≈ 2.77` (хорошо обусловлена)
- Выпуклая квадратичная функция

**Нахождение минимума аналитически:**
```
∂f/∂x₁ = 4x₁ - 2 + x₂ = 0
∂f/∂x₂ = x₁ - 1 + 2x₂ = 0

Решение системы:
x₁ = 3/7, x₂ = 2/7
```
